## 2. 틀린 문제 상세 분석 (중요도 순)

### 🔴 문제44 (복원문제) — 답: `20, 25` → **정답: `25, 20`**
```java
if ((odd && a[i] % 2 != 0) || (!odd && a[i] % 2 == 0)) result += a[i];
...
OE.sum(a, true) + ", " + OE.sum(a, false)
```
- **오류 1 (개념):** 주석에 "홀수 **인덱스**만 더하기"라고 적었으나, 조건은 `a[i] % 2` 로 **값 자체의 홀짝**을 봅니다. 인덱스가 아니라 **원소 값**입니다.
- **오류 2 (순서):** 첫 인자가 `sum(a, true)`(홀수값 합), 둘째가 `sum(a, false)`(짝수값 합).
  - `true` → 1+3+5+7+9 = **25**
  - `false` → 2+4+6+8 = **20**
  - 출력 순서대로 → **`25, 20`**. 두 값을 뒤바꿔 적었습니다.

### 🔴 문제45 (복원문제) — 답: `abcd` → **정답: `dcba`**
```java
String result = rf(str, index-1, seen);  // 먼저 왼쪽(작은 인덱스)부터 처리
if(!seen[c]) { seen[c] = true; return c + result; }  // 현재 문자를 "앞에" 붙임
```
- **개념 오류:** 중복 제거는 맞지만 **조립 순서**를 놓쳤습니다. 재귀가 index 0까지 내려간 뒤 올라오면서, 각 단계가 `c + result`로 **자기 문자를 앞에 prepend** 합니다. 결과적으로 **큰 인덱스 문자가 앞으로** 와서 뒤집힙니다.
- `"abacabcd"` 의 첫 등장 문자는 a,b,c,d 이지만, 조립하면
  `a` → `ba` → `cba` → `dcba` 순으로 쌓여 → **`dcba`**.
- ✅ 기억법: `문자 + 재귀결과` 면 **역순**, `재귀결과 + 문자` 면 **정순**.

### 🟠 문제39 (스택, 빈출 유형) — 답: `213265` → **정답: `213465`**
스택 `into`(push, `++point`) / `take`(pop, `point--`) 추적 문제입니다. 4번째 출력에서 실수했습니다.
```
into(5),into(2)            → [5,2]        point=1
take →2                    → point=0      "2"
into(4),into(1),take →1    → [5,4,1] p=1  "1"
into(3),take →3,take →4    → [5,4,3] p=0  "3","4"   ← 여기서 4를 2로 착각
into(6),take →6,take →5    → [5,6,3] p=-1 "6","5"
```
- **실수 지점:** `into(4)`로 `isWhat[1]`이 2→4로 **덮어써졌는데**, 4번째 pop에서 옛날 값 2가 나온다고 착각. 실제로는 `isWhat[1]=4`가 나옵니다.
- 정답 **`213465`** (이 문제는 실제 기출로도 나온 매우 유명한 스택 문제입니다).

### 🟡 문제31 (꼭확인필요) — 답: `false/false/true/true` → **정답: `true/false/true/true`**
```java
String str1 = "Programming";
String str2 = "Programming";        // 같은 리터럴 → 상수풀에서 같은 객체 공유
String str3 = new String("...");    // new → 힙에 새 객체
str1 == str2   // true  ← 여기가 틀림!
str1 == str3   // false
str1.equals(str3) // true
str2.equals(str3) // true
```
- **개념 오류:** 자바에서 **문자열 리터럴은 String Constant Pool에 캐싱**되어, 같은 리터럴 `"Programming"`은 **동일한 참조**를 가집니다. 따라서 `str1 == str2` 는 **true**.
- `new String(...)`만 매번 새 객체를 만들어 `==`가 false. 값 비교 `equals`는 모두 true.

### 🟡 문제40 (꼭확인필요) — 답: `Ne sd 1` → **정답: `Nd sc 1`**
```c
if(isupper) result = (p[i]-'A'+5) % 25 + 'A';
else if(islower) result = (p[i]-'a'+10) % 26 + 'a';
else if(isdigit) result = (p[i]-'0'+3) % 10 + '0';
```
`"It is 8"` 변환 (핵심: `'t'-'a'`는 **0부터** 세므로 19):

| 문자 | 계산 | 결과 |
|---|---|---|
| I | (8+5)%25 → 13 → 'A'+13 | **N** |
| t | (19+10)%26 → 3 → 'a'+3 | **d** ← 'e'로 오답 |
| (공백) | 그대로 | (공백) |
| i | (8+10)%26 → 18 → 'a'+18 | **s** |
| s | (18+10)%26 → 2 → 'a'+2 | **c** ← 'd'로 오답 |
| (공백) | 그대로 | (공백) |
| 8 | (8+3)%10 → 1 → '0'+1 | **1** |

- **개념 오류:** 알파벳 오프셋을 **1부터(a=1)** 세어 t=20, s=19로 계산했습니다. `문자-'a'`는 **0부터(a=0)** 입니다. → t=19, s=18. 정답 **`Nd sc 1`**.

### ⚪ 문제11 (일반, 기초 중요) — 답: `21 21 30` → **정답: `20 21 30`**
```java
result = j < k ? k++ : --L;   // j=10 < k=20 → 참 → result = k++
```
- **개념 오류:** `k++`는 **후위 증가**. **먼저 현재 값(20)을 result에 대입**한 뒤 k를 21로 증가시킵니다. 주석의 "k++ 실행 후 할당"이 바로 반대로 이해한 부분입니다.
- 따라서 `result = 20`, `k = 21`, `L = 30` → **`20 21 30`**.
- ✅ 후위 `x++` = "쓰고 나서 증가", 전위 `++x` = "증가하고 나서 쓴다".

## 3. 미완성 / 확인 필요 문제

### ⚠️ 문제10 (꼭확인필요) — 첫 줄 답 미확정
```java
int a = 035;   // 앞자리 0 → 8진수 리터럴 (자바·C 공통)
```
- **확정 규칙:** 앞에 `0`이 붙으면 **무조건 8진수**, `0x`가 붙으면 16진수. `035`는 애매하지 않고 **8진수 확정** → `3*8+5 = 29`.
- 정답 3줄: **`29` / `53` / `35`** (b=0x35=53, c=35).

### ⚠️ 문제28 (꼭확인필요) — 답 미작성
- **완전수(perfect number)**: 자기 자신을 제외한 약수의 합 = 자기 자신. 1~100 범위엔 **6**(1+2+3)과 **28**(1+2+4+7+14) 둘뿐입니다.
- `r = 6 + 28 = ` **`34`**. (다음 완전수는 496으로 100 초과 → 6과 28만 외우면 됨)

## 4. ⚠️ 추가 주의 — 문제26 (평가 순서 함정)
```c
printf("%d 그리고 %d", deq(&q), deq(&q));
```
- 큐 값 자체는 2, 3이 맞습니다. 그러나 **C에서 함수 인자 평가 순서는 미정의**입니다.
  - 왼→오 평가: `2 그리고 3` (작성한 답)
  - 오→왼 평가(GCC/MSVC 실제 다수): **`3 그리고 2`**
- 시험에서 한 `printf` 안에 pop/증감이 여러 번 들어가면 **오른쪽부터 계산되는 경우가 많다**는 점을 반드시 기억하세요. 실행하면 `3 그리고 2`가 나올 가능성이 높습니다.

## 5. 개념별 취약점 진단

작성자가 특히 **자주 틀리거나 헷갈리는 개념**을 빈도·중요도순으로 정리했습니다.

1. **증감연산자 (후위/전위) — 최우선 복습**
   - 문제11에서 `k++`를 "증가 후 대입"으로 오해. 기초이자 매 시험 출제.
   - 정리: 후위(`x++`)는 **현재값 사용 → 그 다음 증가**. 전위(`++x`)는 **증가 → 사용**.

2. **재귀 함수의 반환값 조립 순서**
   - 문제45에서 `문자 + 재귀결과`가 역순이 되는 걸 놓침.
   - 정리: 재귀는 **가장 깊은 호출이 먼저 완성**됨. `c + rf(...)` = 역순, `rf(...) + c` = 정순.

3. **자료구조 상태 추적 (스택/큐/연결리스트)**
   - 문제39(스택)에서 "값이 덮어써진 것"을 놓쳐 옛날 값으로 계산.
   - 정리: push/pop마다 **배열·포인터 상태를 표로 그리며** 추적. 연결리스트/큐(18,23,25,26,29)는 잘 풀었으므로, **스택의 index 갱신**만 더 조심.

4. **Java String 비교 (`==` vs `equals`, 상수풀)**
   - 문제31에서 리터럴 interning을 몰라 `str1==str2`를 false로 판단.
   - 정리: **리터럴끼리는 같은 참조(true)**, `new String`은 항상 새 객체(false). `==`는 주소, `equals`는 값.

5. **아스키 문자 연산 (시저 암호류)**
   - 문제40에서 `문자-'a'`를 1부터로 계산. 실제로는 **0부터**(a=0, z=25).
   - 정리: `'문자' - 'a'` = 0-기준 순번. 자주 나오는 유형이니 손으로 표 만들어 연습 권장.

6. **진법 리터럴**
   - 문제10에서 앞자리 `0` = 8진수임을 확신 못함.
   - 정리: `0nnn`=8진수, `0xnnn`=16진수, 그 외 10진수.

7. **평가 순서 미정의 (C)** — 문제26. 한 표현식에 부작용(side effect)이 여러 번이면 순서 주의.
