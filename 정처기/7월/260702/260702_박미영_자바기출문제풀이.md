### ENUM(열거형)

- 정해진 선택지 목록(상수들의 집합)을 안전하게 표현하는 타입
- 값이 몇 개로 딱 고정된 경우에 “문자열/숫자” 대신 쓰는 전용 타입
- 예를 들어, “요일”은 MON, TUE, … 처럼 선택지가 정해져 있을 때 enum이 딱 맞음.
- **enum을 쓰면 뭐가 좋음 ?**
    1. 오타 / 잘못된 값이 원천 차단됨
        - “Mon” / “MONDAY” 같은 문자열은 실수하기 쉬움
        - enum Day (MON, TUE, …) 는 정해진 값만 들어갈 수 있음
    2. 의미가 분명해짐 (가독성이 올라감.)
        - status = 1 같은 매직 넘버보다 status = Status.ACTIVE가 훨씬 명확
- values() : 열거형의 모든 상수를 배열로 반환
- name() : 열거형 상수의 이름을 문자열로 반환 (ex. Tri.A.name() → “A”)

### 싱글톤 디자인 패턴

- 클래스의 인스턴스를 오직 하나만 생성하고, 그 인스턴스를 어디서든 접근할 수 있게 해주는 디자인 패턴
- 클래스의 인스턴스를 하나만 생성하고 싶을 때 사용함.
- 인스턴스는 오직 하나만 생성되고, 프로그램 어디에서든 Connection.get()을 호출하면 같은 인스턴스를 받을 수 있음.

### 비교 연산자

1. **기본형 타입**
    - int, Boolean 등의 경우에는 그 값이 같은 지 비교
    - ex) int a = 1; int b = 1 일 때, a와 b는 같음.
2. **객체 타입**
    - 배열, String, 사용자 정의 클래스 등의 경우에는 == 연산자로 비교할 때 메모리 주소를 비교
    - ex) Person p1 = new Person(”홍길동”); Person p2 = new Person(”홍길동”);
    - p1 == p2는 false가 됨.
    - 메모리 주소 안에서 두 개의 객체가 생성되었고, 그 두 객체 주소는 다름.

### static

- 자바 프로그래밍에서 ‘static’을 사용하면, 그 요소는 특정 객체에 속하지 않고, 해당 클래스의 모든 객체가 공유함.
- 서로 다른 객체라도 static으로 선언된 변수는 그 클래스의 모든 인스턴스에서 공유되어, 어떤 객체가 그 값을 변경하면 다른 모든 객체에서도 그 변경된 값을 볼 수 있음.
- ‘static’ 메소드도 클래스의 특정 인스턴스가 아니라 클래스 자체에 속해 있기 때문에, 객체를 생성하지 않고도 호출할 수 있음.

### Thread와 Runnable 인터페이스

```java
class Car implements Runnable {
    int a;

    public void run() {
        System.out.println("message");
    }
}

public class Main {
    public static void main(String[] args) {
        Thread t1 = new Thread(new Car());
        t1.start();
    }
}
```

- **`implements Runnable`**
    - 뜻 : Car는 Runnable 규칙을 따를게요.
    - `Runnable`은 자바에 원래 있는 인터페이스고, 이걸 쓰면 `run()` 메서드를 반드시 만들어야 함.
- **`public void run()`**
    - `run()`은 `Runnable`을 구현할 때 반드시 써야 하는, 원래 정해진 메서드 이름
- **`Thread`**
    - 자바에 원래 있는 클래스
    - `run()` 안의 일을 실제로 실행해 주는 담당자
- **`Thread t1 = new Thread(new Car())`**
    - 아래처럼 두 줄로 나누면 쉬움.
    
    ```java
    Car car = new Car();  // Car 객체 생성
    Thread t1 = new Thread(car);  // Car의 run()을 실행할 Thread 생성 
    ```
    
    - `Thread`에게 “Car 안에 있는 `run()`을 실행해 줘.” 라고 맡김.
- **`start()`**
    - `Thread` 클래스에 원래 있는 메서드
    - 뜻 : 새로운 스레드를 시작하고, 그 안에서 `run()`을 실행해라.

### 자바 예외 처리

- **예외(Exception)**
    - 프로그램을 실행하다가 발생하는 문제 상황
    - 예외 처리를 안 하면 프로그램이 중간에 멈출 수 있음.
    - 그래서 자바에서는 `try`, `catch`를 써서 “문제가 생기면 이렇게 처리해”라고 적어 둠.
- **`try`**
    
    ```java
    try {
    		func();
    }
    ```
    
    - `try` 안에는 예외가 발생할 가능성이 있는 코드를 넣음.
    - 뜻 : 여기 코드를 실행해 보고, 예외가 생기면 catch로 가자.
- **`catch`**
    
    ```java
    catch (NullPointerException e) {
        sum = sum + 1;
    }
    ```
    
    - 예외를 잡는 곳
    - 뜻 : NullPointerException이 발생하면 이 중괄호 안의 코드를 실행해.
    - `e`  : 발생한 예외를 담는 변수 이름. 보통 오류 내용을 확인할 때 쓸 수 있음.
- `finally`
    
    ```java
    finally {
        sum = sum + 100;
    }
    ```
    
    - 마지막에 실행할 코드를 적는 곳
    - 예외가 발생해도 실행되고, 예외가 안 발생해도 실행됨.
- `throw`
    
    ```java
    throw new NullPointerException();
    ```
    
    - 예외를 직접 발생시키는 문법
    - 뜻 : 지금 NullPointerException을 일부러 발생시킬게.
    - `new NullPointerException()` : `NullPointerException`이라는 예외 객체를 하나 만듦.
    - `throw` : 그 예외를 실제로 발생시킴.
- `throws`
    
    ```java
    static void func() throws Exception {
    ```
    
    - `throws`는 메서드 이름 뒤에 쓰고, ‘이 메서드를 실행하는 중에 예외가 발생할 수 있습니다.’ 라고 미리 알리는 문법
- **`throw`와 `throws` 차이**
    - `throw` : 예외를 직접 발생시킴. 메서드 내부에서 사용
    - `throws` : 이 메서드에서 예외가 발생할 수 있다고 미리 알림. 메서드 선언 부분에서 사용

### 비트 연산자

- **다음 Java 코드에 대한 알맞는 출력값을 쓰시오.**

```java
class Connection {
		private static Connection _inst = null;
		private int count = 0;
		
		static public Connection get() {
				if (_inst == null) {
						_inst = new Connection();
						return _inst;
				}
				return _inst;
		}
		
		public void count() {
				count++;
		}
		
		public int getCount() {
				return count;
		}
}

public class testcon {
		public static void main(String[] args) {
				Connection conn1 = Connection.get();
				conn1.count();
				Connection conn2 = Connection.get();
				conn2.count();
				Connection conn3 = Connection.get();
				conn3.count();
				
				System.out.print(conn1.getCount());
		}
}
```

- 정답
    
    3
    
- 풀이
    - **싱글톤 디자인 패턴**
        - `new Connection()`을 여러 번 못 하게 하고, 처음 만든 Connection 객체 하나를 모두 같이 쓰게 하려는 것
        - `Connection`이라는 물건을 딱 하나만 만들고, `conn1`, `conn2`, `conn3` 이 세 변수가 모두 그 같은 물건 하나를 같이 쓰게 함.
        - 그래서 `conn1.count()`를 해도, `conn2.count()`를 해도, 결국 같은 `count`가 증가함.
    
    ```java
    class Connection {
    
    		// [1] Connection 객체를 저장해 둘 "공용 변수"
    		// private = 이 클래스 안에서만 쓸 수 있게 막는 것 
    		// static = Connection 객체가 없어도 클래스 전체에서 하나만 존재함
    		// _inst = 변수 이름 
    		// null = 아직 Connection 객체를 만든 적이 없음
    		private static Connection _inst = null;
    		
    		// [2] Connection 객체 안에 들어 있는 숫자
    		// 처음 Connection 객체가 만들어질 때 0으로 시작함 
    		private int count = 0;
    		
    		// [3] Connection 객체를 가져오는 메서드
    		// static = 객체가 없어도 Connection.get()처럼 클래스 이름으로 바로 호출 가능
    		static public Connection get() {
    		
    				// [4] _inst가 null인지 확인
    				// 뜻 : "아직 Connection 객체를 만든 적이 없는가?"
    				if (_inst == null) {
    				
    						// [5] 처음 한 번만 Connection 객체를 생성함
    						// new Connection() = Connection 객체 하나 만들기 
    						_inst = new Connection();
    						return _inst;
    				}
    				
    				// [6] 방금 만들었거나, 이미 있던 Connection 객체를 돌려줌
    				// return = get()을 호출한 곳으로 값을 보내는 것 
    				return _inst;
    		}
    		
    		// [7] count를 1 증가시키는 메서드 
    		public void count() {
    				count++;
    		}
    		
    		// [8] 현재 count 값을 알려주는 메서드 
    		public int getCount() {
    				return count;
    		}
    }
    
    public class testcon {
    		public static void main(String[] args) {
    		
    				// [9] Connection.get() 호출
    				// 처음 호출이므로 _inst가 null임
    				// 그래서 Connection 객체가 처음으로 딱 1개 만들어짐 
    				Connection conn1 = Connection.get();
    				
    				// [10] conn1이 가리키는 Connection 객체의 count를 1 증가
    				conn1.count();
    				
    				// [11] 다시 Connection.get() 호출
    				// 이번에는 이미 Connection 객체가 만들어져 있음
    				// 새로 만들지 않고 기존 객체를 그대로 줌 
    				Connection conn2 = Connection.get();
    				
    				// [12] conn2도 conn1과 같은 객체를 가리킴
    				// 따라서 같은 count가 증가함
    				conn2.count();
    				
    				// [13] 
    				Connection conn3 = Connection.get();
    				// [14]
    				conn3.count();
    				
    				// [15] conn1이 가리키는 객체의 count 출력
    				// conn1, conn2, conn3이 사실 같은 객체를 보고 있으므로 3 출력 
    				System.out.print(conn1.getCount());
    		}
    }
    ```
    
    - `private static Connection _inst = null;`
        - `private` : 이 클래스 안에서만 쓸 수 있게 막는 것
            - 이렇게 하면 `Connection` 클래스 밖에서는 `_inst` 를 직접 못 건드림.
        - `static` : 객체마다 따로 생기는 게 아니라, 클래스 전체가 하나만 같이 쓰는 것
            - `static`이 없으면
            
            ```java
            class Connection {
            		Connection _inst;
            }
            ```
            
            - 이렇게 하면 Connection 객체를 여러 개 만들었을 때, 각각 `_inst`를 따로 가짐.
            - `static`을 붙이면 `_inst`는 Connection 객체 안에 여러 개 생기는 변수가 아니라, `Connection` 클래스가 공용으로 하나만 갖는 변수

            		case 값2:

    		public Person(String val) {
    				name = val;
    		}
    		public static String get() {  // Person 클래스의 static 메소드 
    				return name;

    - `static` 메소드에서는 static(정적) 변수만 사용 가능함. 멤버 변수는 사용 불가능.
    - 멤버 변수 : 클래스 안에 선언된 변수. 클래스의 객체가 생성될 때마다 해당 객체와 연결되어 있음.
    - **왜 `static` 메소드에서 멤버 변수는 사용 불가능할까?**

        - **`Person`의 `name`은 언제 할당되는가 ?**
            - `Person a = Person("kim");`
            - 이렇게 `Person` 객체를 생성될 때, `name`의 값이 할당됨.
        - `static` 메소드는 객체 생성 없이도 사용할 수 있다 했는데
        - 그런 `static` 메소드 안에서 객체가 생성되어야만 값이 할당되는 멤버 변수를 사용하려고 하면 당연히 오류가 날 수 밖에 없음.
