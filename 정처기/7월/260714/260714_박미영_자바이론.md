# **Java**

### == 연산자

- 기본 데이터 타입(ex. int, float, char 등)의 값을 비교할 때 자주 사용됨.
- **두 값이 같은지 확인**
- 참조 데이터(ex. 객체)에 대해서 ==를 사용하면, 그것들이 **같은 메모리 주소를 가리키고 있는지** 확인

```java
String text1 = new String("hello");
String text2 = new String("hello");
System.out.println(text1 == text2);

>>> false  // 왜냐하면 메모리 주소가 다르기 때문
```

### equals 메서드

- **객체의 내용이 같은지**를 확인하는 데 사용됨.

```java
String text1 = new String("hello");
String text2 = new String("hello");
System.out.println(text1.equals(text2));

>> true  // 내용이 같기 때문
```

### for문

- 일정한 규칙에 따라 반복해서 명령을 실행할 때 사용함.
- 세 부분으로 구성되어 있음.
    - **초기화**
        - 반복을 시작하기 전에 처음에 설정해야 하는 부분
        - 반복할 때 사용할 변수 설정
    - **조건**
        - 반복을 계속할지 말지를 결정하는 조건
        - 이 조건이 참이면 반복을 계속하고, 거짓이면 반복을 멈춤.
    - **증감**
        - 반복할 때마다 실행되는 부분
        - 보통 반복하는 변수를 증가시키거나 감소시키는데 사용됨.

```java
for(int i = 0; i < 5; i++){
		System.out.println("반복 중: " + i);
}

>>> 반복 중: 0
>>> 반복 중: 1
>>> 반복 중: 2
>>> 반복 중: 3
>>> 반복 중: 4
```

- `int i = 0;`  : 반복을 시작하기 전에 변수 i를 0으로 초기화
- `i < 5` : i가 5보다 작은 동안에는 반복을 계속함. i가 5에 도달하면 반복을 멈춤.
- `i++`  : 매 반복마다 i의 값을 1씩 증가시킴.

### 단순 while문

- 특정 조건이 참인 동안 계속해서 코드 블록을 실행하게 함.
- 조건이 거짓이 되는 순간, 반복이 멈춤.

```java
int i = 0;  // 'i'라는 변수를 0으로 시작함.
while (i < 5) {  // 'i'가 5보다 작은 동안 계속 반복
		System.out.println("i의 값은 " + i + "입니다.");  // 현재 'i'의 값을 출력
		i++;  // 그리고 'i'를 1씩 증가시킴.
}

```

- **for문과의 차이**
    - while 문은 조건이 참인 동안 계속 반복되는데, 주로 반복 횟수가 명확하지 않을 때 사용
    - for 문은 반복 횟수가 정해져 있거나, 반복을 위한 변수의 시작과 끝이 명확할 때 주로 사용됨.

- **continue**
    - continue를 만나면, 반복문의 현재 반복을 즉시 종료하고 다음 반복으로 넘어감.
    - 반복문의 나머지 부분을 건너뛰고, 반복문의 시작 부분으로 돌아가 새로운 반복을 시작함.
- **break**
    - 반복문을 완전히 끝내는 역할
    - break를 만나면, 그 즉시 반복문이 종료되고 반복문 바깥의 다음 코드로 넘어감.

```java
for (int i = 0; i < 10; i++) {
		if (i == 5) {
				continue;  // i가 5일 때, 여기서 나머지 반복문을 건너뛰고 다음 반복으로 넘어감.
		}
		if (i == 8) {
				break;  // i가 8이 되면, for 문을 완전히 끝냄.
		}
}
```

### 상속

- 부모 클래스가 가진 특성(변수)과 행동(메서드)을 자식 클래스가 물려받는 것

```java
class Parent {
		void sayHello() {
				System.out.println("Hello, I'm a parent!");
		}
}

class Child extends Parent {
		// Child 클래스는 Parent 클래스의 sayHello 메서드를 상속받음.
}
```

- Child 클래스는 Parent 클래스로부터 sayHello 메서드를 상속받으니, Child 객체를 만들어 sayHello를 호출하면 “Hello, I’m a parent!” 라는 메시지가 출력됨.

### 오버라이딩

- 상속받은 메서드를 자식 클래스에서 재정의하는 것
- 자식 클래스가 상속받은 메서드를 마음에 들게 바꾸고 싶을 때 사용

```java
class Child extends Parent {
		void sayHello() {
				System.out.println("Hello, I'm a child!");
		}
}
```

- Child 클래스는 Parent의 sayHello 메서드를 오버라이딩해서 “Hello, I’m a chlid!” 라는 새 메시지를 출력하도록 변경

### 오버로딩

- 같은 이름의 메서드를 여러 개 가지되, 매개변수의 유형이나 개수를 다르게 하는 것
- 같은 일을 하는 메서드라도 조금씩 다른 상황에서 사용할 수 있음.

```java
class MathOperations {
		int sum(int a, int b) {
				return a + b;
		}
		
		// 오버로딩된 sum 메서드, 매개변수의 개수가 다름.
		int sum(int a, int b, int c) {
				return a + b + c;
		}
}
```

- MathOperations 클래스에는 sum이라는 메서드가 두 개 있음.
- 하나는 두 숫자를 더하고, 다른 하나는 세 숫자를 더함.
- 매개변수가 다르기 때문에 두 메서드는 서로 다른 버전으로 존재할 수 있음.

### this 키워드

- this는 현재 객체, 즉 메서드나 생성자를 호출하는 객체를 가리키는 데 사용됨.
- 같은 이름의 클래스 변수와 메서드 매개변수가 충돌할 때 이를 구별하기 위해 사용되기도 함.
- 클래스 내의 다른 생성자를 호출할 때도 this()를 사용할 수 있음.

```java
class Flower {
		private String name;  // Flower 객체가 가지고 있는 인스턴스 변수, 즉 객체의 속성. 
		
		Flower(String name) {
				this.name = name;  // 여기서 'this.name'은 클래스 변수를, 'name'은 생성자의 매개변수를 가리킴.
		}
		
		void printName() {
				System.out.println(this.name);  // 'this'로 클래스 변수 'name'에 접근함.
		}
}
		
```

- 여기서 this.name은 클래스의 name 변수를 참조하고, 단순히 name은 생성자로 전달된 매개변수를 참조함.
- 예를 들어서, `Flower flower1 = new Flower("장미");` 로 객체를 하나 만들면
    - `flower1`이라는 `Flower` 객체가 만들어짐.
    - 생성자 안의 `this`는 바로 flower1 객체를 가리킴.
    - `"장미"` 가 매개변수 `name`에 들어감.
    - `this.name = name;` : 현재 만들어지고 있는 Flower 객체의 `name`에, 생성자로 전달받은 `name` 값을 저장해라.
    - `flower1.name = "장미";`

### super 키워드

- super는 부모 클래스를 가리키는 데 사용되며, 부모 클래스의 변수나 메서드, 생성자에 접근할 때 사용함.
- 자식 클래스에서 오버라이드한 메서드가 부모 클래스의 메서드를 호출하고 싶을 때 super를 사용함.

```java
class Plant {
		String type = "Plant";
		
		void printType() {
				System.out.println(this.type);  // 'this'로 자신의 type 변수에 접근함.
		}
}

class Flower extends Plant {
		String type = "Flower";
		
		void printType() {
				super.printType();  // 'super'로 부모 클래스이 printType 메서드를 호출
				System.out.println(this.type);  // 'this'로 자신의 type 변수에 접근  
```

### abstract

- 추상 클래스
- 직접적으로 객체를 만들 수는 없지만, 다른 클래스들이 상속받아 완성해야 하는 기본 틀을 제공함.
- 추상 메서드를 포함할 수 있음.
- 추상 메서드는 실행할 구체적인 코드가 없고, 그저 메서드의 이름, 입력값, 반환 타입만 정의되어 있음.
- 상속받는 클래스는 이 추상 메서드들을 반드시 구현(완성)해야 함.
- 예를 들어, 모든 차에는 주행 기능이 있지만, 각기 다른 차의 주행 방식은 조금씩 다를 수 있음.
- 이 때 ‘차’를 추상 클래스로 생각하고, ‘주행’ 기능을 추상 메서드로 두면, 각 차는 자신의 방식대로 ‘주행’ 메서드를 구현하게 됨.

```java
abstract class Car {
		abstract void drive();  // 추상 메서드: '어떻게 주행할지'는 여기서 정의하지 않음.
		
		void startEngine() {
				System.out.println("엔진이 켜졌어요!");  // 일반 메서드: 모든 차가 공통적으로 사용해요.
		}
}

class Sedan extends Car {
		void drive() {
				System.out.println("매끄럽게 도로를 주행해요!");  // '주행'에 대한 구체적인 구현이 여기에 와요.
		}
}

class Truck extends Car {
		void drive() {
				System.out.println("강력하게 짐을 싣고 달려요!");  // 트럭에 맞게 '주행'을 다르게 구현해요.
		}
}
```

- 여기서 Car는 추상 클래스로서, drive 메서드를 추상 메서드로 가지고 있음.
- Sedan과 Truck은 Car 클래스를 상속받아 drive 메서드를 각자의 방식으로 구현함.
- Car 클래스는 직접 new Car()로 객체를 만들 수 없지만, Sedan과 Truck은 실제 사용 가능한 차들이 됨.

### 인터페이스

- 추상 클래스와 정말 유사하게 ‘구현되지 않은’ 메서드를 미리 선언하고, 이를 상속받은 클래스에서 완성해야 함.
- 사실, interface의 개념을 모르더라도 크게 문제 풀이에 지장은 없음.
- 그냥 추상 클래스와 비슷하게 인터페이스를 상속받은 클래스는 인터페이스가 가진 메소드를 구현해야 한다는 개념만 알고 넘어가도 됨.
- 쉽게 말하면 “이 기능은 반드시 있어야 한다” 는 약속 목록
- 예를 들어 `Vehicle`이라는 인터페이스가 있다고 하면
    
    ```java
    interface Vehicle {
    		void move();
    }
    ```
    
    - `move()`를 어떻게 구현할지는 안 적고, 그냥 “Vehicle을 구현하는 클래스는 `move()` 메서드를 반드시 만들어야 해”라고 약속만 정해둠.
- **추상 클래스와의 차이점**
    1. **상속 개수**
        - 클래스는 부모 클래스를 하나만 상속할 수 있음.
            - `class Child extends Parent { }`
        - 인터페이스는 여러 개를 구현할 수 있음.
            - `class SmartPhone implements Camera, Phone, Internet {
                 // Camera, Phone, Internet의 약속을 모두 구현
             }`
        - 서로 다른 기능을 여러 개 붙이고 싶을 떄 인터페이스를 씀.
    2. **추상 클래스는 “공통 기반”, 인터페이스는 “기능 약속”**
        
        ```java
        abstract class Animal {
        		String name;  // 공통 데이터
        		
        		void eat() {
        				System.out.println("먹는다.");
        		}
        		
        		abstract void sound();
        }
        ```
        
        - `Animal`은 동물들이 공통으로 가지는 이름, 먹는 기능 등을 미리 제공할 수 있음.
        - 그래서 서로 비슷한 종류의 클래스가 공통 기반을 공유할 때 추상 클래스를 씀.
        
        ```java
        interface Flyable {
            void fly();
        }
        ```
        
        - `Flyable`은 “날 수 있다”는 기능 약속일 뿐.
        
        ```java
        class Bird implements Flyable {
            public void fly() {
                System.out.println("새가 날아갑니다.");
            }
        }
        
        class Airplane implements Flyable {
            public void fly() {
                System.out.println("비행기가 날아갑니다.");
            }
        }
        ```
        
        - 새, 비행기, 드론은 서로 완전히 다른 종류지만 모두 날 수 있음.

### ENUM(열거형)

- 정해진 선택지 목록(상수들의 집합)을 안전하게 표현하는 타입
- 값이 몇 개로 딱 고정된 경우에 “문자열/숫자” 대신 쓰는 전용 타입
- 예를 들어, “요일”은 MON, TUE, … 처럼 선택지가 정해져 있을 때 enum이 딱 맞음.
- **enum을 쓰면 뭐가 좋음 ?**
    1. 오타 / 잘못된 값이 원천 차단됨
        - “Mon” / “MONDAY” 같은 문자열은 실수하기 쉬움
        - enum Day (MON, TUE, …) 는 정해진 값만 들어갈 수 있음
    2. 의미가 분명해짐 (가독성이 올라감.)
        - status = 1 같은 매직 넘버보다 status = Status.ACTIVE가 훨씬 명확
- values() : 열거형의 모든 상수를 배열로 반환
- name() : 열거형 상수의 이름을 문자열로 반환 (ex. Tri.A.name() → “A”)

### 싱글톤 디자인 패턴

- 클래스의 인스턴스를 오직 하나만 생성하고, 그 인스턴스를 어디서든 접근할 수 있게 해주는 디자인 패턴
- 클래스의 인스턴스를 하나만 생성하고 싶을 때 사용함.
- 인스턴스는 오직 하나만 생성되고, 프로그램 어디에서든 Connection.get()을 호출하면 같은 인스턴스를 받을 수 있음.

### 비교 연산자

1. **기본형 타입**
    - int, Boolean 등의 경우에는 그 값이 같은 지 비교
    - ex) int a = 1; int b = 1 일 때, a와 b는 같음.
2. **객체 타입**
    - 배열, String, 사용자 정의 클래스 등의 경우에는 == 연산자로 비교할 때 메모리 주소를 비교
    - ex) Person p1 = new Person(”홍길동”); Person p2 = new Person(”홍길동”);
    - p1 == p2는 false가 됨.
    - 메모리 주소 안에서 두 개의 객체가 생성되었고, 그 두 객체 주소는 다름.

### static

- 자바 프로그래밍에서 ‘static’을 사용하면, 그 요소는 특정 객체에 속하지 않고, 해당 클래스의 모든 객체가 공유함.
- 서로 다른 객체라도 static으로 선언된 변수는 그 클래스의 모든 인스턴스에서 공유되어, 어떤 객체가 그 값을 변경하면 다른 모든 객체에서도 그 변경된 값을 볼 수 있음.
- ‘static’ 메소드도 클래스의 특정 인스턴스가 아니라 클래스 자체에 속해 있기 때문에, 객체를 생성하지 않고도 호출할 수 있음.

### Thread와 Runnable 인터페이스

```java
class Car implements Runnable {
    int a;

    public void run() {
        System.out.println("message");
    }
}

public class Main {
    public static void main(String[] args) {
        Thread t1 = new Thread(new Car());
        t1.start();
    }
}
```

- **`implements Runnable`**
    - 뜻 : Car는 Runnable 규칙을 따를게요.
    - `Runnable`은 자바에 원래 있는 인터페이스고, 이걸 쓰면 `run()` 메서드를 반드시 만들어야 함.
- **`public void run()`**
    - `run()`은 `Runnable`을 구현할 때 반드시 써야 하는, 원래 정해진 메서드 이름
- **`Thread`**
    - 자바에 원래 있는 클래스
    - `run()` 안의 일을 실제로 실행해 주는 담당자
- **`Thread t1 = new Thread(new Car())`**
    - 아래처럼 두 줄로 나누면 쉬움.
    
    ```java
    Car car = new Car();  // Car 객체 생성
    Thread t1 = new Thread(car);  // Car의 run()을 실행할 Thread 생성 
    ```
    
    - `Thread`에게 “Car 안에 있는 `run()`을 실행해 줘.” 라고 맡김.
- **`start()`**
    - `Thread` 클래스에 원래 있는 메서드
    - 뜻 : 새로운 스레드를 시작하고, 그 안에서 `run()`을 실행해라.

### 자바 예외 처리

- **예외(Exception)**
    - 프로그램을 실행하다가 발생하는 문제 상황
    - 예외 처리를 안 하면 프로그램이 중간에 멈출 수 있음.
    - 그래서 자바에서는 `try`, `catch`를 써서 “문제가 생기면 이렇게 처리해”라고 적어 둠.
- **`try`**
    
    ```java
    try {
    		func();
    }
    ```
    
    - `try` 안에는 예외가 발생할 가능성이 있는 코드를 넣음.
    - 뜻 : 여기 코드를 실행해 보고, 예외가 생기면 catch로 가자.
- **`catch`**
    
    ```java
    catch (NullPointerException e) {
        sum = sum + 1;
    }
    ```
    
    - 예외를 잡는 곳
    - 뜻 : NullPointerException이 발생하면 이 중괄호 안의 코드를 실행해.
    - `e`  : 발생한 예외를 담는 변수 이름. 보통 오류 내용을 확인할 때 쓸 수 있음.
- `finally`
    
    ```java
    finally {
        sum = sum + 100;
    }
    ```
    
    - 마지막에 실행할 코드를 적는 곳
    - 예외가 발생해도 실행되고, 예외가 안 발생해도 실행됨.
- `throw`
    
    ```java
    throw new NullPointerException();
    ```
    
    - 예외를 직접 발생시키는 문법
    - 뜻 : 지금 NullPointerException을 일부러 발생시킬게.
    - `new NullPointerException()` : `NullPointerException`이라는 예외 객체를 하나 만듦.
    - `throw` : 그 예외를 실제로 발생시킴.
- `throws`
    
    ```java
    static void func() throws Exception {
    ```
    
    - `throws`는 메서드 이름 뒤에 쓰고, ‘이 메서드를 실행하는 중에 예외가 발생할 수 있습니다.’ 라고 미리 알리는 문법
- **`throw`와 `throws` 차이**
    - `throw` : 예외를 직접 발생시킴. 메서드 내부에서 사용
    - `throws` : 이 메서드에서 예외가 발생할 수 있다고 미리 알림. 메서드 선언 부분에서 사용

### 비트 연산자

- **|(OR)**
    - 두 조건 중 하나라도 참이면 참이 되는 연산자
- **^(XOR)**
    - 두 조건이 서로 다를 때만 참이 되는 연산자
- **!(NOT)**
    - 조건의 반대를 나타내는 연산자
