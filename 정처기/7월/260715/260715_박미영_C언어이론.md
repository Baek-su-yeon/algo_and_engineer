# C언어

### 숫자 출력

- `%d` : 10진수
- `%o` : 8진수
- `%x` : 16진수, 소문자
- `%X` : 16진수, 대문자

### main()

- 정처기 실기에서 나오는 일반적인 C 프로그램은 `main()`이 있어야 하고, 실행은 `main()`부터 시작한다고 생각하면 됨.

### 함수

1. **함수 선언**

```c
int r1() {
		return 4;
}
```

- `int` : 이 함수가 정수 값을 돌려준다는 뜻
- `r1` : 함수 이름
- `()` : 함수를 호출할 때 쓰는 표시
- `{ }` : 함수의 실행 내용
- `return 4;` : 함수 실행 결과로 `4`를 돌려줌

1. **함수 호출**

```c
r10()
```

- 뜻 : `r10` 함수를 실행해라.

### `printf()`

- 가장 많이 쓰는 출력 함수
- 기본 형태 : `printf("출력할 내용");`
- 변수도 같이 출력하려면 **서식 지정자** 사용
    
    ```c
    %d   // 정수
    %c   // 문자 하나
    %s   // 문자열
    %f   // 실수
    ```
    
- `printf()`로 변수 값을 출력하려면 무조건 서식 지정자 써야 함.
- **줄바꿈 : `\n`**
    
    ```c
    printf("Hello\n");
    printf("World");
    ```
    
- `%`는 서식 지정자를 시작하는 기호라서, 그냥 쓰면 안 됨.
`%`를 출력하려면 `%%`를 써야 함.

### `puts()`

- 문자열 출력 후 자동 줄바꿈

```c
#include <stdio.h>

int main() {
    puts("Hello");
    puts("World");
    return 0;
}
```

- `puts("Hello");`  = `printf("Hello\n");`
    - 차이점 : `puts()` 는 문자열 출력에만 쓰고, `%d`  같은 서식 지정자는 못 씀.
        
        ```c
        int a = 10;
        
        puts(a);           // 불가능
        printf("%d\n", a); // 가능
        ```
        

### `putchar()`

- 문자 하나 출력

```c
#include <stdio.h>

int main() {
    putchar('A');
    return 0;
}

>>> A
```

- 변수에 들어 있는 문자도 출력할 수 있음.

```c
#include <stdio.h>

int main() {
    putchar('A');
    return 0;
}

>>> B
```

### `printf`, `puts`, `putchar` 차이

| 함수 | 출력 대상 | 줄바꿈 |
| --- | --- | --- |
| `printf()` | 문자, 문자열, 정수, 실수 등 모두 가능 | 직접 `\n` 작성 |
| `puts()` | 문자열만 가능 | 자동 줄바꿈 |
| `putchar()` | 문자 하나만 가능 | 자동 줄바꿈 없음 |

### 포인터

- 변수의 위치를 가리키는 화살표
- 아래 네 개가 제일 중요

```c
&a    // a가 저장된 위치(주소)
p     // 주소를 저장하는 변수
*p    // p가 가리키는 위치에 들어 있는 실제 값
p++   // p를 다음 칸으로 이동
```

### 1. 변수는 값뿐 아니라 “저장된 위치”도 있음

```c
int a = 10;
```

- 이렇게 만들면 컴퓨터 메모리 어딘가에 `a`를 위한 칸이 생기고, 그 안에 `10`이 들어감.

```c
주소 100번지라고 가정

100번지
┌────┐
│ 10 │  ← a
└────┘
```

- `a`는 그 칸에 들어 있는 값인 `10`을 뜻함.
- `&a`는 `a`가 저장된 위치, 즉 주소를 뜻함.

```c
a   = 10
&a  = 100번지
```

### 2. 포인터는 주소를 저장하는 변수

```c
int a = 10;
int* p = &a;
```

- `p`는 int형 변수의 주소를 저장하는 포인터이고, `a`의 주소를 넣음.

```c
a
주소 100번지
┌────┐
│ 10 │
└────┘

p
주소 200번지
┌─────┐
│ 100 │  ─────→ a가 있는 곳
└─────┘
```

- `p`는 `10`을 저장하는 게 아니라, 10이 들어 있는 칸의 위치를 저장하는 거

### 3. `*p`는 “p가 가리키는 곳의 실제 값”

- 위와 같은 상태에서 `*p`는 `p`가 가리키는 위치의 실제 값을 뜻함.
- `p`는 `a`의 주소를 갖고 있으니까 `*p==a`라고 생각하면 됨.
- `*p`로 원래 변수 값도 바꿀 수 있음

```c
int a = 10;
int* p = &a;

*p = 30;  // p가 가리키는 칸에 30을 넣어라
```

- 결국 `a = 30`과 같은 결과가 됨.

### 4. `*`가 두 가지 역할을 해서 헷갈림

- **선언할 때 `*`**
    
    ```c
    int* p;
    ```
    
    - 뜻 : p는 int형 변수의 주소를 저장하는 포인터
- **사용할 때 `*`**
    
    ```c
    *p
    ```
    
    - 뜻 : p가 가리키는 주소에 있는 실제 값

### 5. `&` 도 두 가지 상황이 있음

- `&a` : 변수 앞에 붙으면 a의 주소를 구한다는 뜻
- `a & b` : 숫자나 변수 사이에 있으면 비트 AND 연산

### 6. 배열과 포인터의 관계

```c
int arr[3] = {10, 20, 30};
```

- `arr`은 대부분의 상황에서 배열 첫 번째 칸의 주소처럼 사용됨.
- `arr`은 `&arr[0]`과 같은 위치를 뜻함.

### 7. 배열에서 포인터 이동

```c
int arr[3] = {10, 20, 30};
int* p = arr;
```

| 코드 | 뜻 | 값 |
| --- | --- | --- |
| `p` | `arr[0]`의 주소 | 첫 칸 위치 |
| `*p` | `arr[0]`의 값 | 10 |
| `p + 1` | 다음 칸 주소 | `arr[1]` 위치 |
| `*(p + 1)` | 다음 칸의 값 | 20 |
| `p + 2` | 세 번째 칸 주소 | `arr[2]` 위치 |
| `*(p + 2)` | 세 번째 칸 값 | 30 |
- `arr[1] == *(arr + 1)`

### 8. `p++`는 “다음 요소로 이동”

### 이중 포인터 `**`

- 포인터의 주소를 저장하는 변수

```c
int a = 10;
int* p = &a;
int** pp = &p;
```

- 이걸 단계별로 보면

```c
pp → p → a → 10
```

| 표현 | 의미 |
| --- | --- |
| `a` | 실제 값 `10` |
| `p` | `a`의 주소 |
| `*p` | `a`의 실제 값 `10` |
| `pp` | `p`의 주소 |
| `*pp` | `p` 자체, 즉 `a`의 주소 |
| `**pp` | 최종 실제 값 `10` |
- 즉, `**pp = 30;`을 하면 결국 `a`가 바뀜.
- `**pp`는 사실 `*(*p)`  ⇒ 한 번 따라가서 포인터를 얻고, 한 번 더 따라가서 실제 값을 얻음.

### `sizeof`

- 메모리 크기를 알려주는 연산자
- `sizeof(int)` : `int` 하나가 메모리를 몇 바이트 차지하는지
- 자주 보는 형태
    
    ```c
    sizeof(int)     // int 하나의 크기
    sizeof(char)    // char 하나의 크기
    sizeof(double)  // double 하나의 크기
    sizeof(int*)    // int 주소 하나를 저장하는 포인터의 크기
    ```
    

### `malloc`

- 실행 중에 메모리 공간을 빌리는 함수
- 프로그램이 실행되는 도중에 필요한 만큼 메모리 공간을 새로 만드는 함수
- 사용하려면 `#include <stdlib.h>` 가 필요함.
- **기본 형태**
    
    ```c
    포인터변수 = malloc(필요한_메모리_크기);
    ```
    
- 예를 들어, 정수 한 개를 저장할 공간을 새로 빌리려면
    
    ```c
    int* p = malloc(sizeof(int));
    ```
    
    - 뜻 : 정수 하나를 저장할 수 있는 메모리 공간을 만들고, 그 시작 주소를 `p`에 저장한다.
- 포인터로 값을 넣을 수 있음.
    
    ```c
    *p = 10;
    
    printf("%d", *p);  // 10
    ```
    
- **사용하는 이유**
    - 보통 배열은 만들 때 크기를 미리 정해야 함.
    - ex) `int arr[10];` : 정수 10개짜리 배열
    - 그런데 프로그램을 실행하기 전에는 몇 칸이 필요한지 모르는 경우가 있음.
    - 예를 들어, 사용자에게 크기를 입력받는다면
        
        ```c
        int n;
        scanf("%d", &n);
        ```
        
    - 그 뒤에 `n` 개짜리 배열이 필요할 수 있음.
        
        ```c
        int* arr = malloc(sizeof(int) * n);
        ```
        
    - 이러면 실행 중에 `n`개짜리 배열 공간을 만들 수 있음.
- **`malloc` 결과는 꼭 확인하는 게 좋음**
    - 메모리를 못 만들 수도 있어서, 실제 코드에서는 아래와 같이 확인함.
    
    ```c
    int* p = malloc(sizeof(int));
    
    if (p == NULL) {
        // 메모리 할당 실패
    }
    ```
    
    - 정처기에서는 보통 `malloc`이 성공했다고 가정하고 문제를 내는 경우가 많지만, 개념으로는 알아두면 좋음.

### `free`

- 빌린 메모리를 반납하는 함수
- `malloc()`으로 만든 공간은 다 쓰고 나면 `free()`로 반납해야 함.

```c
int* p = malloc(sizeof(int));

*p = 10;

free(p);
```

- 뜻 : `p`가 가리키던 메모리 공간은 이제 더 이상 쓰지 않겠다. 반납한다.
- 보통 `free` 뒤에 아래와 같이 처리함.

```c
free(p);
p = NULL;
```

- 왜냐하면 `free(p)`를 했다고 해서 `p` 안의 주소값이 자동으로 사라지는 건 아니기 때문
- **`malloc`와 `free`는 세트**
    - 아래 흐름이 기본
        
        ```c
        int n = 5;
        int* arr = malloc(sizeof(int) * n);
        
        /* arr 사용 */
        
        free(arr);
        arr = NULL;
        ```
        

### 정적 변수

1. 메모리에 한 번만 할당됨.
    - 정적 변수는 프로그램이 실행될 때 메모리에 할당하고, 프로그램이 종료될 때까지 그 값을 유지
    - 그래서 함수가 여러 번 호출되더라도 새로 초기화되지 않음.
2. 초기화는 한 번만 이루어짐.
    - 정적 변수를 선언할 때 초기값을 주면, 프로그램 실행 시 한 번만 초기화됨.
    - 이후에는 이전에 저장된 값을 그대로 사용함.

### 구조체

- 여러 가지 다른 종류의 변수를 하나로 묶어 새로운 형태의 ‘데이터 뭉치’를 만드는 것
- 예를 들어, 친구 한 명이 있으면 이 친구에 대해 이름, 나이, 전화번호 등 다양한 정보가 있음.
- C언어의 구조체를 사용하면 이런 정보들을 한데 모아 ‘친구’ 라는 하나의 구조체로 만들 수 있음.
