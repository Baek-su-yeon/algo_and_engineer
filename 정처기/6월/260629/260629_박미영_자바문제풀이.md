### super 키워드

- super는 부모 클래스를 가리키는 데 사용되며, 부모 클래스의 변수나 메서드, 생성자에 접근할 때 사용함.
- 자식 클래스에서 오버라이드한 메서드가 부모 클래스의 메서드를 호출하고 싶을 때 super를 사용함.

```java
class Plant {
		String type = "Plant";
		
		void printType() {
				System.out.println(this.type);  // 'this'로 자신의 type 변수에 접근함.
		}
}

class Flower extends Plant {
		String type = "Flower";
		
		void printType() {
				super.printType();  // 'super'로 부모 클래스이 printType 메서드를 호출
				System.out.println(this.type);  // 'this'로 자신의 type 변수에 접근  
```

### abstract

- 추상 클래스
- 직접적으로 객체를 만들 수는 없지만, 다른 클래스들이 상속받아 완성해야 하는 기본 틀을 제공함.
- 추상 메서드를 포함할 수 있음.
- 추상 메서드는 실행할 구체적인 코드가 없고, 그저 메서드의 이름, 입력값, 반환 타입만 정의되어 있음.
- 상속받는 클래스는 이 추상 메서드들을 반드시 구현(완성)해야 함.
- 예를 들어, 모든 차에는 주행 기능이 있지만, 각기 다른 차의 주행 방식은 조금씩 다를 수 있음.
- 이 때 ‘차’를 추상 클래스로 생각하고, ‘주행’ 기능을 추상 메서드로 두면, 각 차는 자신의 방식대로 ‘주행’ 메서드를 구현하게 됨.

```java
abstract class Car {
		abstract void drive();  // 추상 메서드: '어떻게 주행할지'는 여기서 정의하지 않음.
		
		void startEngine() {
				System.out.println("엔진이 켜졌어요!");  // 일반 메서드: 모든 차가 공통적으로 사용해요.
		}
}

class Sedan extends Car {
		void drive() {
				System.out.println("매끄럽게 도로를 주행해요!");  // '주행'에 대한 구체적인 구현이 여기에 와요.
		}
}

class Truck extends Car {
		void drive() {
				System.out.println("강력하게 짐을 싣고 달려요!");  // 트럭에 맞게 '주행'을 다르게 구현해요.
		}
}
```

- 여기서 Car는 추상 클래스로서, drive 메서드를 추상 메서드로 가지고 있음.
- Sedan과 Truck은 Car 클래스를 상속받아 drive 메서드를 각자의 방식으로 구현함.
- Car 클래스는 직접 new Car()로 객체를 만들 수 없지만, Sedan과 Truck은 실제 사용 가능한 차들이 됨.

### 인터페이스

- 추상 클래스와 정말 유사하게 ‘구현되지 않은’ 메서드를 미리 선언하고, 이를 상속받은 클래스에서 완성해야 함.
- 사실, interface의 개념을 모르더라도 크게 문제 풀이에 지장은 없음.
- 그냥 추상 클래스와 비슷하게 인터페이스를 상속받은 클래스는 인터페이스가 가진 메소드를 구현해야 한다는 개념만 알고 넘어가도 됨.
- 쉽게 말하면 “이 기능은 반드시 있어야 한다” 는 약속 목록
- 예를 들어 `Vehicle`이라는 인터페이스가 있다고 하면
    
    ```java
    interface Vehicle {
    		void move();
    }
    ```
    
    - `move()`를 어떻게 구현할지는 안 적고, 그냥 “Vehicle을 구현하는 클래스는 `move()` 메서드를 반드시 만들어야 해”라고 약속만 정해둠.
- **추상 클래스와의 차이점**
    1. **상속 개수**
        - 클래스는 부모 클래스를 하나만 상속할 수 있음.
            - `class Child extends Parent { }`
        - 인터페이스는 여러 개를 구현할 수 있음.
            - `class SmartPhone implements Camera, Phone, Internet {
                 // Camera, Phone, Internet의 약속을 모두 구현
             }`
        - 서로 다른 기능을 여러 개 붙이고 싶을 떄 인터페이스를 씀.
    2. **추상 클래스는 “공통 기반”, 인터페이스는 “기능 약속”**
        
        ```java
        abstract class Animal {
        		String name;  // 공통 데이터
        		
        		void eat() {
        				System.out.println("먹는다.");
        		}
        		
        		abstract void sound();
        }
        ```
        
        - `Animal`은 동물들이 공통으로 가지는 이름, 먹는 기능 등을 미리 제공할 수 있음.
        - 그래서 서로 비슷한 종류의 클래스가 공통 기반을 공유할 때 추상 클래스를 씀.
        
        ```java
        interface Flyable {
            void fly();
        }
        ```
        
        - `Flyable`은 “날 수 있다”는 기능 약속일 뿐.
        
        ```java
        class Bird implements Flyable {
            public void fly() {
                System.out.println("새가 날아갑니다.");
            }
        }
        
        class Airplane implements Flyable {
            public void fly() {
                System.out.println("비행기가 날아갑니다.");
            }
        }
        ```
        
        - 새, 비행기, 드론은 서로 완전히 다른 종류지만 모두 날 수 있음.

### [기출 예제] 23년 3회 실기

- **다음 Java 코드이다. 올바른 출력 결과를 작성하시오.**

```java
class Parent {
		int compute(int num) {
				if(num <= 1)
						return num;
				return compute(num-1) + compute(num-2);
		}
}

class Child extends Parent {
		int compute(int num) {
				if (num <= 1)
						return num;
				return compute(num-1) + compute(num-3);
		}
}

public class main {
		public static void main(String args[]) {
				Parent obj = new Child();
				System.out.print(obj.compute(7));
		}
}
```

- 정답
    
    2
    
- 풀이
    - num ≤ 1 ⇒ return num
    - num(2) = num(1) + num(-1) = 0
    - num(3) = num(2) + num(0) = 0
    - num(4) = num(3) + num(1) = 1
    - num(5) = num(4) + num(2) = 1
    - num(6) = num(5) + num(3) = 1
    - num(7) = num(6) + num(4) = 2

```java
public class main {
		public static void main(String args[]) {
				Parent obj = new Child();
				System.out.print(obj.compute(7));
		}
}
```

- 자식 클래스 Child 객체를 부모 클래스 Parent에 할당
- 껍데기는 부모 클래스이지만, 자식 객체가 할당되었으므로, compute 메소드는 자식클래스인 Child 객체의 메소드로 동작함.
- **이런 짓을 왜 함?**
    - **부모 타입으로 공통 기능을 다루면서도, 실제 객체마다 다른 동작을 하게 만들기 위해서**
    - **객체지향에서 ‘다형성’ 이라고 함.**
    - **`Parent` = 변수를 바라보는 기준, 사용할 수 있는 기능의 범위**
    - **`new Child()` = 실제로 만들어진 객체**
    - **겉으로는 Parent로 다루지만 실제 내부에는 Child 객체가 들어 있음.**

### [기출 예제] 25년 1회 실기

- **다음은 Java 코드에 대한 문제이다. 아래 코드를 확인하여 알맞는 출력값을 작성하시오.**

```java
public class Main {
		public static void main(String[] args) {
				new Child();
				System.out.println(Parent.total);
		}
}

class Parent {
		static int total = 0;
		int v = 1;
		
		public Parent() {
				total += (++v);
				show();
		}
		
		public void show() {
				total += total;
		}
}

class Child extends Parent {
		int v = 10;
		
		public Child() {
				v += 2;
				total += v++;
				show();
		}
		
		@Override
		public void show() {
				total += total * 2;
		}
}
```

- 정답
    
    54 
    
- 풀이
    - `new Child();` 에서 Child를 생성했는데 왜 Parent도 실행되어야 함?
    - **무조건 생성자는 부모 → 자식 순으로**
    - **생성자** : 클래스 이름과 같음 / 반환형이 없음 / void도 안 씀 / `new 클래스명()` 을 하면 자동 실행됨
    - `new Child()`로 만든 객체 안에는 실제로 아래와 같은 공간이 함께 존재함.
        
        ```java
        Child 객체
         ├─ Parent가 가진 v
         └─ Child가 새로 가진 v
        ```
        
        - `Child` 객체를 완성하려면, 먼저 부모 영역부터 제대로 초기화해야 함.
        - 그 규칙 때문에 자식 생성자는 시작하자마자 부모 생성자를 먼저 호출
    - `Child` 생성자에는 실제로 아래 코드가 숨어 있다고 생각하면 됨.
        
        ```java
        public Child() {
            super();  // 컴파일러가 자동으로 넣음
            v += 2;
            total += v++;
            show();
        }
        ```
        
        - 내가 작성하지 않아도 자바가 자동으로 맨 첫 줄에 넣음.
        - `super()`는 부모 생성자를 뜻함.
    - `show()`는 `Parent.show()`가 아니라 `Child.show()`가 실행됨.
        - 왜냐하면 실제 생성 중인 객체가 `Child` 객체이기 때문
        - **메서드는 오버라이딩되어 있으면 실제 객체 기준으로 호출됨.**

    - `static` 메서드는 객체마다 따로 있는 메서드가 아니라, 클래스에 소속된 메서드
    - 겉보기에는 `ref`가 실제로 Child 객체니까 `Child.id()`가 호출될 것 같음.
    - 하지만 static 메서드는 오버라이딩 규칙을 적용하지 않음.
    - `ref`의 실제 객체가 Child인지 보지 않고, ref라는 변수의 선언 타입을 봄.
    - `ref` 는 `Parent` 타입으로 선언되었으므로, `Parent.id()`로 판단됨.
    - **외우기**
        
        ```java
        일반 메서드 → 실제 객체 타입 기준
        static 메서드 → 변수의 선언 타입 또는 클래스명 기준
        ```

### [기출 예제] 25년 2회 실기

- **다음은 Java 코드에 대한 문제이다. 아래 코드를 확인하여 알맞는 출력값을 작성하시오.**

```java
public class Main {
		
		static interface F {
				int apply(int x) throws Exception;
		}
		
		public static int run(F f) {
				try {
						return f.apply(3);
				} catch (Exception e) {
						return 7;
				}
		}
		
		public static void main(String[] args) {
				
				F f = (x) -> {
						if (x > 2) {
								throw new Exception();
						}
						return x * 2;
				};
				
				System.out.print(run(f) + run((int n) -> n + 9));
		}
}
```

- 정답
    
    19
    
- 풀이
    
    ```java
    public class Main {
    		
    		// 1. 함수형 인터페이스 F
    		// apply(int x)라는 메서드 하나를 반드시 구현해야 함.
    		// 이 메서드는 int를 반환하고, Exception을 던질 수도 있음. 
    		static interface F {
    				int apply(int x) throws Exception;
    		}
    		
    		// 2. F 타입의 객체(람다식)를 받아 실행하는 메서드 
    		public static int run(F f) {
    				try {
    						// 3. 전달받은 람다식의 apply(3)을 실행함.
    						// 즉, 어떤 람다를 넣든 x 또는 n 자리에 3이 들어감. 
    						return f.apply(3);
    						
    				} catch (Exception e) {
    						// 4. apply(3) 실행 중 예외가 발생하면
    						// 여기로 와서 7을 반환함
    						return 7;
    				}
    		}
    		
    		public static void main(String[] args) {
    				
    				// 5. 첫 번째 람다식 생성
    				// f.apply(x)를 호출하면 아래 코드가 실행됨. 
    				F f = (x) -> {
    				
    						// run(f)를 호출하면 run 내부에서 f.apply(3)을 실행하므로
    						// 여기서 x는 3이 됨. 
    						if (x > 2) {  // 3 > 2 -> true
    								
    								// 예외 발생
    								// return x *2;는 실행되지 않고 run(의 catch로 이동함. 
    								throw new Exception();
    						}
    						return x * 2;
    				};
    				
    				// 6. 첫 번째 run(f) 실행
            //    run 안에서 f.apply(3) 호출
            //    x = 3
            //    3 > 2 이므로 Exception 발생
            //    catch문이 실행되어 7 반환
            //
            //    따라서 run(f)의 결과는 7
            
            
            // 7. 두 번째 run() 실행
            //    (int n) -> n + 9 라는 람다식을 바로 전달함.
            //
            //    run 안에서 apply(3)을 호출하므로
            //    n = 3
            //
            //    n + 9 = 3 + 9 = 12
            //
            //    예외가 없으므로 그대로 12 반환
            //
            //    따라서 run((int n) -> n + 9)의 결과는 12
            
            // 8. 최종 계산
            //    run(f) = 7
            //    run((int n) -> n + 9) = 12
            //
            //    7 + 12 = 19 출력
    		
    				System.out.print(run(f) + run((int n) -> n + 9));
    		}
    }
    ```
    
    - `run((int n) -> n + 9))`
        - 아까의 `f` 를 다시 실행하는 게 아님.
        - 새 람다를 하나 만들어서 `run()`에 전달한 거임.
    - 위 람다는 아래의 새 객체와 같음.
        
        ```java
        F temp = new F() {
            public int apply(int n) {
                return n + 9;
            }
        };
        ```
