### 02 Python 문제1 - 예제 1.py

```python
x, y = input('입력 :').split('-')
a = [ 'abc123', 'def456', 'ghi789' ]
a.append(x)
a.append(y)
a.remove('def456')
print(a[1][-3:], a[2][:-3], sep = ',')
for i in range(3, 6):
	print(i, end = ' ')
```

```python
# 정답
# 입력 "abcdef-xyz999"로 가정
789,abc
3 4 5 
```

<aside>

**오답 풀이**

이 문제는 ① remove 후 인덱스가 당겨진다는 점, ② 출력에 쓰이는 게 y가 아니라 x라는 점을 헷갈리게 하는 트랩

입력값을 '-' 기준으로 split → x, y로 언패킹 (예: "abcdef-xyz999" → x='abcdef', y='xyz999')

리스트 변화 추적
→ a = ['abc123', 'def456', 'ghi789']
→ append(x) → ['abc123', 'def456', 'ghi789', 'abcdef']
→ append(y) → ['abc123', 'def456', 'ghi789', 'abcdef', 'xyz999']
→ remove('def456') → ['abc123', 'ghi789', 'abcdef', 'xyz999']

여기서 def456이 빠지면서 인덱스가 한 칸씩 당겨짐
→ a[1] = 'ghi789' → a[1][-3:] = '789'  (입력과 무관하게 항상 '789' 고정)
→ a[2] = 'abcdef' (append한 x) → a[2][:-3] = 'abc'  (뒤 3글자 잘라냄)
→ print(..., sep=',') → "789,abc"

마지막 반복문 range(3, 6) = [3, 4, 5], end=' ' → "3 4 5 " (끝에 공백 한 칸)

※ a[1][-3:]는 무조건 '789'이고, 두 번째 값만 입력 x에 따라 달라짐. y는 출력에 안 쓰임

</aside>

### 03 Python 문제2 - 예제 1.py

```python
a = {'apple', 'lemon', 'banana'}
a.update( {'kiwi', 'banana'} )
a.remove('lemon')
a.add('apple')
for i in a:
	print("과일명 : %s" % i)
```

```python
# 예상 정답
# set이라 순서는 보장 안 됨
과일명 : apple
과일명 : kiwi
과일명 : banana
```

<aside>

**내 풀이**

set은 중복을 허용하지 않는 자료형

→ update({'kiwi', 'banana'}) → kiwi 추가, banana는 이미 있어서 무시 → {apple, lemon, banana, kiwi}
→ remove('lemon') → lemon 제거 → {apple, banana, kiwi}
→ add('apple') → 이미 있어서 무시 → {apple, banana, kiwi}

최종 원소 3개. set은 순서를 보장하지 않으므로 출력 순서는 달라질 수 있음

</aside>

### 기출 따라잡기 - 문제 4.py

```python
a = 100
result = 0
for i in range(1,3):
	result = a >> i
	result = result + 1
print(result)
```

```python
# 정답
26
```

<aside>

**오답 풀이**

result를 누적(+=)하는 게 아니라, 매 반복마다 result = a >> i 로 새로 덮어쓰는 게 함정. 결국 마지막 반복의 값만 남음

>> 는 우측 시프트 연산(2의 거듭제곱으로 나눈 몫)
→ range(1, 3) = [1, 2]
→ i=1 : result = 100 >> 1 = 50 → result = 50 + 1 = 51  (다음 반복에서 덮어써짐)
→ i=2 : result = 100 >> 2 = 25 → result = 25 + 1 = 26

100 >> 2 = 100 // 4 = 25 → 최종 result = 26
(51을 더하는 식으로 누적하면 오답)

</aside>

### 기출 따라잡기 - 문제 5.py

```python
class CharClass:
	a = [ 'Seoul', 'Kyeongi', 'Inchon', 'Daejeon', 'Daegu', 'Pusan'];
myVar = CharClass()
str01 = ' '
for i in myVar.a:
	str01 = str01 + i[0]
print(str01)
```

```python
# 정답
 SKIDDP
```

<aside>

**오답 풀이**

str01의 초기값이 빈 문자열 ''이 아니라 공백 한 칸 ' '인 게 함정. 결과 맨 앞에 공백이 붙음

각 원소의 첫 글자 i[0]만 이어붙임
→ Seoul → S
→ Kyeongi → K
→ Inchon → I
→ Daejeon → D
→ Daegu → D
→ Pusan → P

str01 = ' ' + S + K + I + D + D + P = " SKIDDP"
(Daejeon, Daegu 둘 다 D라 D가 두 번, 그리고 앞 공백 빼먹으면 오답)

</aside>

### 기출 따라잡기 - 문제 6.py

```python
a = "REMEMBER NOVEMBER"
b = a[0:3] + a[12:16]
c = "R AND %s" % "STR"
print(b + c)
```

```python
# 예상 정답
REMEMBER AND STR
```

<aside>

**내 풀이**

문자열 인덱스
R(0) E(1) M(2) E(3) M(4) B(5) E(6) R(7) ' '(8) N(9) O(10) V(11) E(12) M(13) B(14) E(15) R(16)

→ a[0:3] = 'REM'  (0,1,2)
→ a[12:16] = 'EMBE'  (12,13,14,15)
→ b = 'REM' + 'EMBE' = 'REMEMBE'
→ c = "R AND %s" % "STR" = 'R AND STR'
→ b + c = 'REMEMBE' + 'R AND STR' = 'REMEMBER AND STR'

</aside>

### 기출 따라잡기 - 문제 7.py

```python
lst = [1,2,3]
dst = {i : i* 2 for i in lst}
s = set(dst.values( ))
lst[0] = 99
dst[2] = 7
s.add(99)
print(len(s & set(dst.values( ))))
```

```python
# 정답
2
```

<aside>

**오답 풀이**

각 자료구조가 "언제" 바뀌는지 시점을 따지는 게 핵심. 특히 lst를 바꿔도 이미 만들어진 dst엔 영향이 없다는 점이 함정

→ dst = {1:2, 2:4, 3:6}  (컴프리헨션으로 i:i*2)
→ s = set(dst.values()) = {2, 4, 6}
→ lst[0] = 99 → lst는 [99,2,3]이 되지만 dst는 이미 완성됐으므로 변화 없음 (트랩)
→ dst[2] = 7 → dst = {1:2, 2:7, 3:6}  (키 2의 값만 교체)
→ s.add(99) → s = {2, 4, 6, 99}

마지막 줄
→ set(dst.values()) = {2, 7, 6}
→ s & set(dst.values()) = {2,4,6,99} & {2,6,7} = {2, 6}  (& 는 교집합)
→ len({2, 6}) = 2

</aside>

### 기출 따라잡기 - 문제 8.py

```python
x, y = input("x, y의 값을 공백으로 구분하여 입력 : ").split(' ')
print("x의 값 :", x)
print("y의 값 :", y)
```

```python
# 예상 정답 (입력 "3 5" 가정)
x의 값 : 3
y의 값 : 5
```

<aside>

**내 풀이**

input()으로 받은 문자열을 split(' ')로 공백 기준 분리한 뒤 x, y에 언패킹

입력이 "3 5"라면
→ split(' ') → ['3', '5']
→ x = '3', y = '5'  (문자열 그대로, 숫자 아님)

print에서 콤마(,) 구분자는 자동으로 공백 한 칸을 넣어줌
→ "x의 값 : 3", "y의 값 : 5"

※ 공백으로 구분된 토큰이 정확히 2개여야 함. 1개나 3개면 언패킹 개수가 안 맞아 ValueError 발생

</aside>
