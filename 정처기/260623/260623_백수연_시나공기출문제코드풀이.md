# 정처기 실기 공부

<aside>

## 핵심 정리

이번 파일에서 제일 중요한 개념은 이거야.

```
Parentp=newChild();
```

이런 형태가 나오면 무조건 두 가지를 나눠서 봐야 해.

1. **호출 가능한 메서드인지 확인할 때는 변수 타입 기준**
    - `Parent`에 있는 메서드만 바로 호출 가능
2. **실제로 어떤 메서드가 실행되는지는 실제 객체 기준**
    - 실제 객체가 `Child`이고, Child가 그 메서드를 오버라이딩했다면 Child 메서드 실행
</aside>

### 02 Java 문제 - 예제 1.java

```java
abstract class Animal {
	String a = " is animal";
	abstract void look();
	void show() {
		System.out.println("Zoo");
	}
}
class Chicken extends Animal {
	Chicken() {
		look();
	}
	void look() {
		System.out.println("Chicken" + a);
	}
	void display() {
		System.out.println("two wings");
	}
}
public class Test {
	public static void main(String[] args) {
		Animal a = new Chicken();
		a.show();
	}
}
```

```java
// 예상 정답
Chicken is animal
Zoo
```

<aside>

**풀이**

Chicken은 Animal을 상속 받는 자식 객체

Animal a = new Chicken();을 통해 생성자 Chicken() 동작 → look() 실행, **Chicken 객체 안에 상속받은 Animal의 필드 `a`가 들어있다** → a.show(); 실행

</aside>

### 기출 따라잡기 - 문제 1.java

```java
class Parent {
	void show() { System.out.println("parent"); }
}
class Child extends Parent {
	void show() {System.out.println("child"); }
}
public class Test {
	public static void main(String[ ] args) {
		Parent pa = new Child();
		pa.show();
	}
}
```

```java
// 예상 정답
parent
```

<aside>

**풀이**

Parent 형식으로 생성된 객체 pa니까 Parent 클래스의 show() 함수 실행

**오답풀이**

오버라이딩된 메서드는 **실제 객체 기준**으로 실행

즉, 변수 타입은 Parent여도 실제 만들어진 객체인 Child의 show() 메서드가 실행됨
그래서 child가 출력됨

</aside>

### 기출 따라잡기 - 문제 2.java

```java
abstract class Vehicle {
	String name;
	abstract public String getName(String val);
	public String getName() {
		return "Vehicle name : " + name;
	}
}
class Car extends Vehicle {
	private String name;
	public Car(String val) {
		name = super.name = val;
	}
	public String getName(String val) {
		return "Car name : " + val;
	}
	public String getName(byte[] val) {
		return "Car name : " + val;
	}
}
public class Test {
	public static void main(String[] args) {
		Vehicle obj = new Car("Spark");
		System.out.print(obj.getName());
	}
}
```

```java
// 예상 정답
Vehicle name : Spark
```

<aside>

**풀이**

name = [super.name](http://super.name) = val;에서 super.name을 통해 부모 클래스의 name = Spark 할당

Java에서 오버라이딩된 메서드는 실제 객체 기준으로 실행된다.

이 문제에서는 Car가 getName()을 오버라이딩하지 않았기 때문에 Vehicle의 getName() 이 실행된다.

</aside>

### 기출 따라잡기 - 문제 3.java

```java
class Parent {
	int compute(int num) {
		if(num <= 1) return num;
		return compute(num - 1) + compute(num - 2);
	}
}
class Child extends Parent {
	int compute(int num) {
		if(num <= 1) return num;
		return compute(num - 1) + compute(num - 3);
	}
}
public class Test {
	public static void main(String[] args) {
		Parent obj = new Child();
		System.out.print(obj.compute(4));
	}
}
```

```java
// 예상 정답
3
```

<aside>

**풀이**

compute(3) + compute(2)

→ compute(2) + compute(1) + compute(1) + coupute(0)

→ compute(1) + coupute(0) + 1 + 1 + 0

→ 1 + 0 + 1 + 1 + 0 = 3

**오답풀이**

문제 1과 동일한 오버라이딩 문제로 Child의 compute 메서드 실행

compute(3) + coupute(1)

→ coupute(2) + coupute(0) + 1

→ coupute(1) + coupute(-1) + 0 + 1

→ 1 + (-1) + 0 + 1 = 1

</aside>
